# clean-di — modular example

Demonstrates `imports:` — composing a parent context from a reusable
sub-config module (`mathConfig`).

## Structure

```
src/
  MathConfig.ts             — config interface (precision: number)
  Adder.ts                  — add(a, b) with fixed decimal places
  Multiplier.ts             — multiply(a, b) with fixed decimal places
  Calculator.ts             — orchestrates Adder + Multiplier
  mathConfig.di.ts          — sub-config: declares adder + multiplier beans
  CalcContext.di.ts         — top-level context: imports mathConfig, adds calculator
  CalcContext.di.generated.ts — generated by clean-di-codegen
  index.ts                  — usage: get container, run math operations
```

## Run

```bash
pnpm install           # from the monorepo root
pnpm --filter clean-di-example-modular build
node examples/modular/dist/index.js
# → 3.14 + 2.71 = 5.85
# → 2.5 × 1.5 = 3.75
```

## Key concepts

| Concept | Where |
|---------|-------|
| `defineConfig({ beans: {...} })` | Reusable sub-module; does NOT call `defineContext` |
| `imports: [mathConfig]` | Pull sub-config beans into the parent scope |
| Transitive resolution | `calculator: bean(Calculator)` receives `adder` and `multiplier` from mathConfig |
| Synthetic config bean | `precision` field of `MathConfig` auto-wires to `Adder` + `Multiplier` constructors |
| Auto-added imports | Generated file imports `Adder`, `Multiplier` even though they're not in `CalcContext.di.ts` |
