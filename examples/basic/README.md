# clean-di — basic example

Demonstrates the simplest clean-di setup: a single context with two beans
(`Logger` + `Greeter`) and a config bean supplying a `name` string.

## Structure

```
src/
  AppConfig.ts              — config interface (name: string)
  Logger.ts                 — simple logger
  Greeter.ts                — uses Logger + name from config
  GreeterContext.di.ts      — context definition (source of truth)
  GreeterContext.di.generated.ts — generated by clean-di-codegen
  index.ts                  — usage: get container, call greet()
```

## Run

```bash
pnpm install           # from the monorepo root
pnpm --filter clean-di-example-basic build
node examples/basic/dist/index.js
# → [INFO] Hello, World!
# → Hello, World!
```

## Regenerate

```bash
cd examples/basic
pnpm clean-di-codegen     # writes GreeterContext.di.generated.ts
```

## Key concepts

| Concept | Where |
|---------|-------|
| `defineContext<AppConfig>()` | pins config type; `name` field auto-becomes a synthetic bean |
| `bean(Logger)` / `bean(Greeter)` | class-based beans resolved by constructor param types |
| Generated `createContext<AppConfig, { greeter: Greeter }>` | provides the typed `Container` |
| `container.greeter.greet()` | fully typed access — no casts |
